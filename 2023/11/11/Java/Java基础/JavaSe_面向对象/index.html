面向对象

面向对象是一种软件设计思想，它将现实世界的问题抽象为对象。面向对象编程（OOP）是一种基于对象的编码方式，其中程序由对象组成，对象可以通过封装、继承和多态等概念来相互关联。

面向对象的一个原则是封装，它通过将成员变量设置为私有（private），并提供公共的成员方法（public methods）来控制对这些变量的访问。这样可以隐藏内部实现的细节，提高安全性和代码的可维护性。

super 关键字

如何实现的

super 关键字的功能是通过编译器生成的特殊指令来实现的。

  1. 调用父类的方法： 当在子类中使用 super.method() 调用父类的方法时，实际上编译器会生成一个指向父类方法的特殊调用指令。这个指令确保调用的是父类版本的方法，而不是子类中可能存在的同名方法。

  2. 访问父类的字段： 如果在子类中使用 super.field 访问父类的字段，编译器会生成一个指令，使程序能够访问父类的字段。这对于解决子类和父类中存在同名字段的情况非常有用。

  3. 调用父类的构造方法： 在子类的构造方法中使用 super(args) 调用父类的构造方法时，在子类对象初始化时，首先调用父类的构造方法。这是为了保证父类的初始化过程得以执行。

在Java虚拟机（JVM）中，这些指令被翻译成底层的字节码指令，确保正确的方法和字段被调用。实际上，这些细节对于大多数开发者而言是透明的，我们只需关注使用 super 关键字的语法和语义。

如何使用

在Java中，super 关键字主要用于访问父类的成员，包括字段、方法和构造方法。它允许子类在重写父类方法时调用父类的相应方法，以及在子类构造方法中调用父类的构造方法。以下是 super 关键字的主要用途：

  1. 调用父类的方法： 在子类中可以使用 super 调用父类的方法，以便在子类中添加或修改特定的行为而不是完全覆盖父类的方法。

  2. 访问父类的字段： 如果子类和父类有同名的字段，使用 super 关键字可以访问父类中的字段。

  3. 调用父类的构造方法： 在子类的构造方法中，可以使用 super 调用父类的构造方法，以确保父类的初始化工作得以执行。

final 关键字

在Java中，final 是一个关键字，用于表示不可改变的、最终的特性。final 可以用于变量、方法、类等不同的上下文，并且其主要作用有以下几个方面：

  1. 不可变的变量： 当用 final 修饰一个变量时，表示该变量只能被赋值一次。一旦赋值之后，就不能再修改。一般用于常量字符串。

  2. 不可继承的类： 当用 final 修饰一个类时，表示该类不能被其他类继承。

  3. 不可重写的方法： 当用 final 修饰一个方法时，表示该方法不能被子类重写。

  4. 常量： 在接口中，final 用于定义常量，这些常量在实现接口的类中不能被修改。

在Java中，Object 类是所有类的根类。每个类都隐式地继承自 Object 类，因此 Object 类中定义的方法可以在所有对象中使用。以下是一些 Object 类中常用的共有方法：

    通过 Lombok 的 @Data 注解生成的 equals() 方法是重写过的。Lombok 的 @Data 注解整合了 @EqualsAndHashCode 注解，该注解生成的 equals() 方法中，对比了对象的所有字段，而不是仅仅是==

  3. hashCode() 方法：

    • 返回对象的哈希码。默认实现返回对象的内存地址的哈希码。

    哈希码是一个32位的整数，用于提高在哈希表等数据结构中查找对象的效率。

    hashCode() 方法的实际作用是为了提供一个快速的散列值，使得存储在哈希表等数据结构中的对象能够分布得更加均匀。在使用哈希表进行查找时，通过哈希码可以迅速定位可能包含目标对象的桶，从而减少搜索的时间复杂度。

    哈希码是通过哈希算法计算得出的。这个哈希算法通常由Java运行时环境提供，但也可以在自定义类中进行重写。

    关于 hashCode() 方法和哈希算法的一些约定：

    1. 相等的对象必须具有相等的哈希码： 如果两个对象通过 equals() 方法比较是相等的，那么它们的哈希码必须相等。反之则不一定成立，即哈希码相等的对象不一定通过 equals() 方法比较相等。

    2. 不同的对象应该有不同的哈希码： 为了提高哈希表的效率，不同的对象应该有不同的哈希码。然而，由于哈希码是一个有限的整数，哈希码冲突（即不同对象具有相同哈希码）是可能的。

    在实际编程中，如果自定义类需要被放入哈希表等集合中，通常需要同时重写 hashCode() 和 equals() 方法，以确保哈希表的正确性和性能。这两个方法的实现需要保持一致性，即相等的对象必须具有相等的哈希码。

  4. clone() 方法：

    • 创建并返回对象的副本。要使用 clone() 方法，类必须实现 Cloneable 接口，并覆盖 clone() 方法。

    默认是浅拷贝 所以如果字段是对象引用，那么只会复制引用，而不会复制引用指向的对象本身

多态

基本使用

多态（Polymorphism）是面向对象编程的一个重要概念，它允许使用基类Animal的引用来引用子类Dog的对象，Animal animal = new Dog()，从而实现在运行时选择调用哪个类的方法。多态有两种主要形式：编译时多态（静态多态）和运行时多态（动态多态）。

编译时多态（静态多态）：

编译时多态是通过方法的重载实现的，即同类同名不同参。编译器在编译时根据方法的参数类型选择调用哪个方法。

运行时多态（动态多态）：

运行时多态是通过方法的重写（覆盖）和继承实现的。子类可以提供对父类方法的特定实现，并且在运行时，根据对象的实际类型来确定调用哪个方法。

在上述示例中，Animal 类有一个 makeSound() 方法，而 Dog 和 Cat 类分别重写了这个方法。在运行时，通过基类 Animal 的引用，实际对象是 Dog 或 Cat，决定了调用哪个类的 makeSound 方法。

多态是如何实现的

多态的实现涉及到两个主要概念：虚方法表（Virtual Method Table，VMT）和动态绑定（Dynamic Binding）。

虚方法表（Virtual Method Table，VMT）：

在Java中，每个对象都有一个虚方法表，这个表中存储着对象所属类的方法信息。虚方法表是在类加载的时候创建的，并且在运行时动态地更新。

当一个对象被创建时，Java虚拟机为其分配内存并创建虚方法表。当调用对象的方法时，虚拟机通过查找虚方法表来确定实际要调用的方法。这个过程被称为静态绑定，因为在编译时就已经确定了要调用的方法。

动态绑定（Dynamic Binding）：

动态绑定是多态的关键。在运行时，当通过基类的引用调用一个方法时，实际上是根据对象的实际类型来决定调用哪个类的方法。也就是根据new 后面跟的类型确定，比如 Animal animal = new Dog()中的 就确定了调用Dog的方法，而不是 Animal

抽象类

定义

  1. 抽象类（Abstract Class）： 抽象类是不能被实例化的类，通常用于作为其他类的基类。抽象类可以包含抽象方法，也可以包含具体方法。抽象方法必须在抽象类中声明。

实现抽象方法：

抽象方法是一种没有方法体（实现）的方法，它只包含方法的声明。抽象方法用 abstract 关键字来声明，并且只能存在于抽象类或接口中。抽象方法的存在是为了让子类提供具体的实现。就比如在上述例子中，Circle 和 Rectangle 是 Shape 的子类，它们必须实现 Shape 类中的抽象方法 area()。通过实现这个抽象方法，子类提供了具体的行为。而基类只是提供了方法的声明(这也许就是"抽象"?)

使用

在上述例子中，通过创建 Circle 和 Rectangle 对象，使用它们的抽象父类 Shape 的引用。在运行时，具体调用了子类中实现的 area() 方法，实现了多态性。

接口

接口（Interface）定义了一组抽象方法，但没有提供方法的具体实现。接口可以被类实现（implements）

定义

接口使用 interface 关键字定义，方法默认为抽象方法，且可以包含常量(业务上一些常量也是统一放在接口中，不过是专门放常量)和默认方法（Java 8及以后版本）。

实现

类通过 implements 关键字来实现接口，并提供接口中定义的方法的具体实现。

使用

与抽象类的区别

  1. 多继承： 类只能继承一个类，但是可以实现多个接口。也就是所谓的==“单继承，多实现”==，这允许一个类具备多个不同接口的功能，实现更灵活的代码组织。

  2. 构造方法： 接口不能有构造方法，而抽象类可以有构造方法。一个类在实现接口时，没有办法调用接口的构造方法。

  3. 方法实现： 接口中的方法默认是抽象的，而抽象类可以包含抽象方法和非抽象方法。在Java 8及以后版本，接口可以包含默认方法和静态方法。

  4. 字段： 接口中的字段默认是常量，而抽象类中可以包含各种类型的字段。

通常，如果一个类需要提供一些默认的实现并且具备多个不同接口的功能，那么使用接口更为合适。如果类需要有构造方法、字段以及提供一些共享的实现，那么使用抽象类更为合适。在实际开发中，通常会根据具体需求选择使用抽象类、接口，或它们的组合。

比较抽象类和接口

  • 抽象类：
    • 用于表示一般性的类继承关系，强调代码的重用。
    • 提供了一些默认实现，子类可以选择性地覆盖或继承这些实现。

  • 接口：
    • 用于定义一种契约，表示类应该具备的行为或能力。
    • 提供一种多继承机制，使得类可以实现多个接口。

以电子设备的设计为例，使用抽象类和接口来演示其应用。

假设我们要设计一系列电子设备，这些设备可能包括手机、平板电脑和电视。我们想要通过面向对象的设计，为这些设备建立一个类层次结构。

  • 设计基类

在这里，ElectronicDevice 是一个抽象类，表示通用的电子设备，它包含了一些通用的特征和行为。但是，它并没有具体的实现 displayInfo 方法，这个方法由子类去实现。

  • 子类实现并实现抽象方法

Phone、Tablet 和 TV 类是 ElectronicDevice 抽象类的具体子类，它们分别实现了抽象方法 displayInfo。

  • 接口设计以及具体实现

在这里，Connectable 是一个接口，表示设备可以连接到互联网。SmartPhone 类实现了这个接口，同时继承了 Phone 类。

这其中 各种电子设备(手机、电脑、电视)的抽象方法displayInfo -> 通用

而联网接口 -> 赋予更多的专用能力

  • 场景的使用

这个例子中，我们使用了抽象类和接口来设计电子设备的类层次结构。抽象类 ElectronicDevice 提供了通用的特征和行为，而具体的设备类（Phone、Tablet、TV）则通过继承和实现来具体化这些行为。接口 Connectable 则表示设备可以连接到互联网，SmartPhone 类实现了这个接口，同时也继承了 Phone 类。
