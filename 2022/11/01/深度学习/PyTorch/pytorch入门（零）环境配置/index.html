Anaconda是什么

conda是什么

认识Anaconda之前，需要先理解conda，conda与pip npm相同，都是包管理工具，说白了就是用来安装包的。

conda 跟 pip 相比，它可以快速安装、运行和更新软件包及其依赖项

最主要的区别还是在于conda 支持虚拟环境

什么意思呢？假如说一个项目需要Python3.6，但是另外一个项目需要Python3.8，难道要来回删除与安装吗？

可能两个还是不明显，如果有4、5个，那看起来就很麻烦了

但是用conda就可以创建两个虚拟环境，一个环境装3.6，另一个装3.8，这样就解决问题了

这里的虚拟环境可以看成是一个个的电脑

另外一个情况是不同的包之间是有依赖关系的，这里的依赖关系就像我们写两个python文件，1.py需要2.py的函数这种依赖关系一样

而且不仅有依赖关系，版本必须是要一一对应的

可以回忆一下，有多少报错是版本问题导致的呢

而conda就可以自动处理这种依赖关系与包的版本，从而下载合适的包

Anaconda

Anaconda就是在conda的基础上开发出来的

Anaconda功能如下

  1. 内置Python
  2. 拥有强大的包管理工具conda
  3. 可用超过600个Python数据科学库

安装pytorch

理清几个关系

包、虚拟环境、conda（pip）、Python解释器、CUDA、GPU、CPU有啥关系？

  • 虚拟环境类似于计算机
    • 里面有Python解释器（创建环境时候可以指定版本，不指定就是最新版本）
    • 有各种包，像tensorflow、pytorch都是包
    • conda/pip负责管理包的安装、卸载、更新

GPU（Graphics processing unit）图像处理单元

NVIDIA创建了CUDA平台，开发者可以利用CUDA操作NVIDA的显卡，即GPU

因为GPU相比CPU更擅长数据处理，所以会加速深度学习的训练速度

还有一点就是显卡是硬件，我们需要在电脑上安装一个驱动，这样我们才能通过驱动操作显卡

这个驱动就是CUDA driver，而且这个是硬件层面的

还有一个是 CUDA runtime 即软件层面的cuda

我们要保证 cuda runtime version <= cuda driver version

安装流程

查看cuda driver version

终端输入nvidia-smi

说明 cuda driver version = 12.0 我们需要安装 <= 12.0的 cuda runtime version

确认 cuda runtime version

pytorch 官网 给了 11.6 和 11.7的 cuda runtime 所以两个都是可以的

安装pytorch

因为按照官网命令出了问题一直解决不了，所以安装了以前的cuda版本11.3 并且通过 -c 指定清华镜像源

之后等待安装成功即可

确认安装成功
