是什么

Spring 是一个轻量级的控制反转(IoC)和面向切面(AOP)的容器（框架）

为什么用 Spring

  1. Spring 是一个轻量级的框架 , 非侵入式的 .
  2. 控制反转 IoC , 面向切面 Aop
  3. 对事物的支持 , 对框架的支持

Spring Boot 与 Spring Cloud

  • Spring Boot 是 Spring 的一套快速配置脚手架，可以基于 Spring Boot 快速开发单个微服务;
  • Spring Cloud 是基于 Spring Boot 实现的；
  • Spring Boot 专注于快速、方便集成的单个微服务个体，Spring Cloud 关注全局的服务治理框架；
  • 如果你要学习 SpringCloud 必须要学习 SpringBoot。

Spring IOC

IOC 的本质

  1. 控制反转 IOC(Inversion of Control)，是一种设计思想，DI(依赖注入)是实现 IOC 的一种方法
  2. Spring 容器在初始化时先读取配置文件，根据配置文件将对象存入容器中，程序使用时再从 IOC 容器中取出需要的对象。
  3. 采用 XML 方式配置 Bean 的时候，Bean 的定义信息是和实现分离的
  4. 采用注解的方式可以把两者合为一体，Bean 的定义信息直接以注解的形式定义在实现类中，从而达到了零配置的目的。

控制反转是一种通过描述（XML 或注解）并通过第三方去生产或获取特定对象的方式。在 Spring 中实现控制反转的是 IOC 容器，其实现方法是依赖注入（Dependency Injection,DI）

使用注解后 bean 的 id

默认情况下 bean 类名的首字母小写就是 bean 的 id

@AutoWired 的工作流程

  • 根据类型到 IOC 容器中找
    • 能找到唯一 bean，那么直接装配
    • 同类型 bean 有多个(这个可以写一个接口，然后写多个实现类，然后把接口进行装配，这样做来测试这种情况)
      • 默认情况根据 @Autowired 标记成员变量的变量名(userService)作为 bean 的 id(默认是 bean 类名首字母小写, 也就是说 bean 的 id 是可以指定的，但一般不用)进行匹配
        • 能找到，装配
        • 不能，失败
      • 使用 @Qualifier 注解：根据 @Qualifier 注解中指定的名称作为 bean 的 id 进行匹配
        • 能找到，装配
        • 不能，失败

测试同类型 bean 有多个这种情况

  1. 创建接口 UserService
package com.shaohua.api;

public interface UserService {
    void print();
}
  1. 创建 两个实现类
    1. 都加上 @Service 注解
    2. UserServiceImpl2 特别设置 value = “userService”
// UserServiceImpl1
@Service
public class UserServiceImpl1 implements UserService {
    @Override
    public void print() {
        System.out.println("service from UserServiceImpl1");
    }
}

// UserServiceImpl2
@Service(value = "userService")
public class UserServiceImpl2 implements UserService {
    @Override
    public void print() {
        System.out.println("service from UserServiceImpl2");
    }
}
  1. 测试
@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(value = {"classpath:springConfigure.xml"})
public class SpringTest {
    @Autowired
    private UserService userService;


    @Test
    public void test() {
        userService.print();
    }
}
  1. 结果

service from UserServiceImpl2

证明

  1. 默认情况下 @Autowired 标记成员变量的变量名(userService)作为 bean 的 id 进行匹配, 因为 bean 的 id 默认是 bean 类名首字母小写，在本例中就是两个实现类名的首字母小写(userServiceImpl1, userServiceImpl2)
  2. 所以本来会匹配失败, 但是注解中给 value 赋值是可以改变 bean 的默认 id 的， 所以当把 UserServiceImpl2 中 id 通过@Service(value = "userService")改为 userService 就刚好和 @Autowired 标记成员变量的变量名(userService)匹配上了，从而匹配成功

Spring AOP

AOP 初使用

对 AOP 的初步理解

为了避免核心功能代码与不重要功能的代码耦合 ,让开发者把重心放在核心功能的实现上

举个不太恰当的例子。就比如说电竞选手与战队教练的关系，职业选手只需要关注自己能力的提升。

而战队的生活起居、与其他队伍约训练赛之类的事情就交给战队教练去做。

其实 AOP 就是做这个的

  1. 创建 Mapper 以及 Service
@Repository
public class CustomerMapper {
    @Autowired
    private JdbcTemplate jdbcTemplate;

    public void updateCustomerNameById(Integer customerId, String customerName) {
        String sql = "update t_customer set customer_name = ? where customer_id = ?";
        jdbcTemplate.update(sql, customerName, customerId);
    }

    public Customer selectCustomerById(Integer customerId) {
        String sql = "select * from t_customer where customer_id = ?";
        return jdbcTemplate.queryForObject(sql, new BeanPropertyRowMapper<>(Customer.class), customerId);
    }


    public void updateCustomerIdById(Integer customerId, Integer customerNewId) {
        String sql = "update t_customer set customer_id = ? where customer_id = ?";
        jdbcTemplate.update(sql, customerNewId, customerId);
    }
}

@Transactional // 声明式事务 注解
@Service
public class CustomerService {
    @Autowired
    private CustomerMapper customerMapper;

    public Customer selectCustomer(Integer customerId) {
        return customerMapper.selectCustomerById(customerId);
    }

    public void updateCustomerById(Integer customerId, Integer customerNewId, String customerNewName) {
        customerMapper.updateCustomerNameById(customerId, customerNewName);
        customerMapper.updateCustomerIdById(customerId, customerNewId);
    }
}
